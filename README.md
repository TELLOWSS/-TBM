## 2) 로컬 실행

### 사전 준비

- Node.js (LTS 권장)

### 실행 절차

1. 의존성 설치
   - `npm install`
2. API 키 설정
   - `.env.local` 파일에 `GEMINI_API_KEY=...` 입력
3. 개발 서버 실행
   - `npm run dev`

## 3) 화면별 핵심 기능

### A. 통합 대시보드

- 금일 인원/위험요인/문서연계 상태를 한 화면에서 모니터링
- 위험성평가 연계 경보(미연계/동일월 미일치) 즉시 확인
- 정규화 운영 경보(`CRITICAL/WARNING`) 발생 시 심층연구소로 즉시 이동
- 팀별 우선순위 카드 클릭으로 해당 팀 중심 조치 동선 진입

### B. TBM 등록

- 일일 TBM 기록 신규 작성/수정/삭제
- 위험성평가 문서와 연계 정보 연결
- 사진/영상 기반 분석 결과와 현장 입력 데이터를 통합 저장

### C. 위험성평가 관리

- 월간/정기/최초 위험성평가 목록 관리
- 우선 위험요인 및 액션노트 관리
- TBM과 동일월 연계율 확인을 위한 기준 데이터 제공

### D. 문서 보관소

- 기록 조회/검색/필터/인쇄/일괄 삭제
- 연계 상태 필터(전체/미연계/미일치)로 보정 대상 선별
- 초기 필터 배지 기반 진입 및 원터치 해제 지원

### E. 안전데이터 심층연구소

- 다중 팀 필터/기간 필터 기반 분석
- 위험 스펙트럼, 팀 활동 히트맵, 추이 분석
- 미등록 팀 정규화 대기열 운영(요청 → 승인/반려)
- 운영 KPI(대기건수/평균대기/24h 초과/반려사유 Top) 모니터링
- 정규화 이력 기간 필터 및 CSV 내보내기

## 4) 권장 운영 시나리오 (실무 기준)

### 시나리오 1: 오전 점검

1. 대시보드에서 연계 경보/정규화 경보 확인
2. 경보 클릭으로 심층연구소 진입
3. 대기 요청 승인/반려 처리 후 결과 확인
4. 문서 보관소에서 미연계/미일치 잔여건 재점검

### 시나리오 2: 주간 분석

1. 심층연구소에서 기간 `7D` 또는 `30D` 선택
2. 팀별 히트맵으로 취약 팀 식별
3. 위험 상위요인/연계율 추이 확인
4. 필요 시 정규화 이력 CSV 내보내기 후 보고서 첨부

### 시나리오 3: 감사/외부 점검 대응

1. 문서 보관소에서 대상 기간 기록 조회
2. 심층연구소 정규화 이력(누가/언제/무엇/몇 건) 확인
3. 반려 사유 통계로 데이터 품질 개선 항목 도출
4. 백업 파일 생성 후 보관

---

## 5) 역할별 사용 포인트

### 실무자

- TBM 입력 정확도(팀/날짜/연계 정보) 우선
- 미등록 팀 발생 시 정규화 요청 등록
- 문서 보관소 필터로 누락 건 즉시 수정

### 평가자/관리자

- 정규화 승인/반려 기준 일관성 유지
- 24시간 초과 대기건 우선 처리
- 반려 사유 코드 표준화로 재발 방지

### 대외 검토자

- 연계율/이력/반려사유 통계를 근거로 운영 성숙도 판단
- 백업/복구 가능성 및 데이터 연속성 확인

---

## 6) 백업/복구 운영

### 백업

- 전체/일지/TBM/위험성평가 범위 선택 가능
- 전체 백업에는 팀/서명/정규화 로그/정규화 요청 상태 포함

### 복구

- 복구 시 데이터 정합성 검증 로직 적용
- 병합 복구 기준으로 기존 데이터와 충돌 최소화

---

## 7) 자주 발생하는 운영 이슈

- 팀명이 길어 화면이 깨짐
  - 최신 UI는 긴 문자열 줄바꿈/말줄임 처리 적용
- 경보는 보이는데 조치 위치 찾기 어려움
  - 대시보드 경보 클릭 시 심층연구소 조치 구간으로 자동 포커스 이동
- 대기 요청 누적으로 처리 지연
  - KPI 카드의 24h 초과/평균대기 지표 기준으로 우선순위 처리
